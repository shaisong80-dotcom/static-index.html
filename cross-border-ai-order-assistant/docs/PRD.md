# 跨境电商订单异常识别与 AI 自动化处理系统 PRD

## 1. 项目背景

跨境电商订单履约过程中，运营人员需要反复检查订单字段、物流轨迹、SKU、报关金额和订单状态。人工排查耗时长、标准不统一，容易造成发货延迟、清关风险和客服响应滞后。

本项目以 ERP 订单同步后的异常处理场景为背景，设计一个订单异常识别与 AI 辅助处理工具，帮助运营人员快速发现问题、判断优先级并生成处理建议。

## 2. 项目目标

- 将零散订单问题标准化为可识别的异常规则。
- 通过看板展示订单异常率、异常类型分布和高优先级问题。
- 为异常订单生成原因分析、处理动作和客服/运营沟通话术。
- 降低人工排查成本，提高订单履约处理效率。

## 3. 用户角色

| 角色 | 关注点 | 典型操作 |
| --- | --- | --- |
| 运营专员 | 订单是否能正常履约 | 上传订单、筛选异常、跟进处理 |
| 物流负责人 | 物流轨迹和面单是否异常 | 查看物流单号缺失、轨迹未更新 |
| 财务/报关人员 | 金额和申报信息是否合理 | 复核订单金额、报关金额 |
| 客服人员 | 如何向客户解释进度 | 复制 AI 生成的话术 |

## 4. 业务流程

1. ERP 或平台订单导出 CSV。
2. 运营人员上传订单文件。
3. 系统对订单字段进行标准化解析。
4. 异常识别规则检查物流、地址、联系方式、SKU、金额、报关和状态。
5. 系统生成异常看板和订单列表。
6. 运营人员选择异常订单生成 AI 处理建议。
7. 按优先级分派给运营、物流、财务或客服跟进。
8. 处理完成后复盘异常类型，为后续流程优化提供依据。

## 5. 功能需求

### 5.1 订单上传

- 支持上传 CSV 订单文件。
- 支持加载示例订单数据用于演示。
- 系统读取订单字段并进入异常分析流程。

### 5.2 异常识别规则

| 异常类型 | 识别规则 | 优先级 |
| --- | --- | --- |
| 物流单号缺失 | logistics_no 为空 | 高 |
| 收件地址不完整 | address 为空或过短 | 高 |
| 电话格式异常 | phone 为空或不是有效数字格式 | 中 |
| SKU 缺失 | sku 为空 | 高 |
| 数量异常 | quantity 小于等于 0 | 高 |
| 订单金额异常 | order_amount 小于等于 0 | 高 |
| 报关金额偏低 | declared_amount 明显低于 order_amount | 中 |
| 订单物流状态不一致 | 订单取消但物流仍在运输或已签收 | 高 |
| 物流轨迹未更新 | 已发货但物流状态仍为待处理 | 中 |

### 5.3 异常看板

- 展示订单总数。
- 展示异常订单数。
- 展示异常率。
- 展示高优先级问题数量。
- 展示异常类型分布。

### 5.4 异常订单列表

- 展示订单号、平台、国家、金额、订单状态、物流状态。
- 展示异常标签。
- 支持按全部、高优先级、待跟进、正常筛选。
- 支持对单个订单生成 AI 建议。

### 5.5 AI 处理建议

系统根据异常类型生成：

- 异常原因。
- 建议处理动作。
- 处理优先级。
- 建议责任人。
- 客服/运营沟通话术。

## 6. 非功能需求

- 页面加载后默认展示示例数据，方便演示。
- 规则结果需清晰可解释，便于产品和研发沟通。
- 后端接口结构简单，便于后续接入真实 ERP/API。
- 前端适配桌面和常见笔记本屏幕。

## 7. 验收标准

- 可以成功加载示例订单数据。
- 可以上传 CSV 并展示分析结果。
- 异常订单能显示对应异常标签。
- 指标卡片和异常分布能随数据变化。
- 点击订单后能生成 AI 处理建议。
- 简历和面试中可以清楚说明业务背景、需求分析、功能设计和技术实现。
