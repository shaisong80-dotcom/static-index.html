# 面试讲解稿

## 1. 为什么做这个项目

我投递的岗位偏产品需求分析和 AI 自动化方向，岗位里提到跨境电商、ERP、物流追踪、报关核算和大模型自动化。我希望用一个小型 Demo 证明自己不只是会写代码，也能从业务痛点出发，完成需求分析、规则设计、页面原型和技术落地。

## 2. 项目解决什么问题

跨境电商订单履约中，运营需要人工检查物流单号、地址、电话、SKU、金额、报关金额和订单物流状态。如果订单量变大，人工排查效率低，而且不同人判断标准不一致。

这个系统把这些问题转成标准化规则，自动识别异常订单，并生成优先级和处理建议。

## 3. 我的主要工作

我主要负责四部分：

1. 梳理业务流程，输出 PRD 和流程图。
2. 设计异常识别规则，把业务问题转成可执行逻辑。
3. 使用 Python 实现订单解析、异常检测和接口输出。
4. 搭建 Web 看板，展示异常数据，并实现 AI 建议模块。

## 4. 技术实现

后端使用 Python 标准库实现本地服务，读取 CSV 订单数据后进行规则检测。前端使用 HTML、CSS 和 JavaScript 实现看板、筛选和建议展示。

AI 建议模块目前采用规则模板模拟大模型输出，后续可以接入真实大模型 API，根据异常标签、订单字段和历史处理记录生成更自然的建议。

## 5. 项目亮点

- 业务场景贴近跨境电商履约。
- 覆盖物流、地址、SKU、金额、报关、状态一致性等高频异常。
- 不只是写功能，也补充了 PRD、流程图和面试表达。
- 体现了产品需求分析和技术落地结合的能力。

## 6. 如果继续优化

- 接入真实 ERP 或电商平台 API。
- 增加 RPA 自动同步和自动派单。
- 使用大模型 API 生成更灵活的处理建议。
- 增加异常处理闭环，记录处理结果和责任人。
- 根据历史数据分析高频异常来源，反向优化业务流程。
